# R&T Мебель — CMS + сайт (готово к заливке)

Сайт и админка теперь **одно приложение**: сайт отдаётся из `public/`, заявки с сайта
падают прямо в админку. Никаких отдельных хостингов и CORS не нужно.

## Что внутри
- `public/index.html` + `public/data.js` — сайт R&T (форма «Оставить заявку» → в админку)
- `public/admin/…` — панель управления (заявки, товары, отзывы, настройки, статистика…)
- `routes/public.js` — публичный API сайта (без логина): приём заявок, настройки
- `routes/api.js` — админ-API (только после входа)
- `server.js`, `db/init.js` — сервер и база (SQLite)
- `scripts/hash.js` — генерация пароля для админки

## Локальный запуск (для проверки)
```
npm install
node scripts/hash.js мойпароль      # скопируй полученный хеш
# создай .env из .env.example, вставь SESSION_SECRET и ADMIN_PASSWORD_HASH
npm start
```
- Сайт:   http://localhost:3000/
- Админка: http://localhost:3000/admin

## Заливка на Timeweb App Platform (приложение mebel-cms)

1. **Сетевой диск (чтобы база и заявки не стирались при редеплое).**
   Панель → «Сетевые диски» → создать (5–10 ГБ) → примонтировать к `mebel-cms` в `/data`.

2. **Переменные окружения** (App Platform → mebel-cms → Переменные):
   ```
   NODE_ENV=production
   DATA_DIR=/data
   SESSION_SECRET=<длинная случайная строка>
   ADMIN_PASSWORD_HASH=<хеш из node scripts/hash.js>
   # опционально для писем о заявках:
   SMTP_HOST=smtp.gmail.com
   SMTP_PORT=587
   SMTP_USER=...
   SMTP_PASS=...
   NOTIFY_EMAIL=...
   ```

3. **Команды сборки/старта:**  Build `npm install` · Start `npm start` (Node 18/20/22 — ок).

4. **Проверка после деплоя** (на домене приложения):
   - `/` — открывается сайт, жмёшь «Оставить заявку» → заявка появляется в `/admin/leads`;
   - `/admin` — вход по паролю;
   - сделай ещё один редеплой → заявка осталась на месте (диск работает).

## Что изменилось при пересборке (важно)
- Форма сайта раньше била в `/contact` (был закрыт авторизацией → заявки не доходили).
  Теперь шлёт на `POST /api/public/lead` — публично и доходит.
- В корзине вместо нескольких кнопок одна — **«Оставить заявку»**.
  Можно оставить заявку и с пустой корзиной (просто телефон/имя).
- `bcrypt` → `bcryptjs`, `better-sqlite3` → 11.x: убраны проблемы со сборкой при деплое.
- Пароль и секреты больше не зашиты в код — только через переменные окружения.
- База и фото вынесены в `DATA_DIR` — переживают редеплой при подключённом диске. Сессии хранятся в памяти, поэтому после редеплоя нужно заново войти в `/admin`.
